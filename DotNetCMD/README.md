# DotNetCommander

DotNetCommander — двопанельний файловий менеджер у стилі Total Commander / Norton Commander / Double Commander на `Windows Forms` і `.NET 8`.

## Про проєкт

Це робочий Windows-first файловий менеджер з власним `Quick View`, внутрішніми редакторами, асинхронними файловими операціями та commander-style hotkeys.

Поточна активна збірка:
- `Commander.NET.csproj`
- `net8.0-windows10.0.22621.0`
- версія `1.7.0`

## Основні можливості

- дві файлові панелі з адресним рядком, панеллю дисків і commander-style навігацією;
- `Quick View` (`F2` / `Ctrl+Q`) для тексту, Markdown, зображень, RTF і невеликих CSV-подібних файлів;
- вбудоване порівняння файлів через `Shift+F3`: text diff, CSV diff, image compare і binary summary;
- вбудовані `TextEdit`, `RtfEdit`, `CsvView`, `ImageView`;
- `TextEdit` і `RtfEdit` з `Save As...`, `F1`-help і опційним status bar зі статистикою документа;
- `RtfEdit` з Markdown-to-RTF preview, контекстними Markdown-стилями `Normal / H1..H6 / Code` і кастомним preview-styling через `Options`;
- асинхронні `copy`, `move`, `delete` з прогресом, скасуванням і коректним завершенням без unhandled cancellation;
- drag-and-drop з `Explorer`, інших файлових менеджерів і між власними панелями;
- обробка конфліктів під час `copy/move`: `overwrite`, `skip`, `rename`, `apply to all`;
- створення та розпакування архівів `ZIP`, `TAR`, `TAR.GZ` / `TGZ` засобами .NET 8 без сторонніх бібліотек;
- відкриття підтримуваних архівів безпосередньо у файловій панелі як тимчасових логічних пристроїв: пристрій з'являється у drive toolbar на час роботи з архівом і зникає після виходу;
- спільний кеш системних іконок за розширенням/типом, тому каталоги й файли мають ті самі іконки у файловій та архівній панелях;
- commander-style командний рядок під панелями для запуску програм, скриптів і shell-команд із параметрами в каталозі активної панелі; підтримуються історія команд, вставлення поточного імені та persistent console;
- shell-інтеграція: `Open`, `Open with...`, `Properties`;
- `Shift+Enter` для запуску `.bat/.cmd/.exe/.com/.ps1` у консолі, що не закривається;
- `Ctrl+R` для ручного перечитування активної файлової панелі;
- `Ctrl+N` для створення нового файла через `SaveFileDialog`;
- `Shift+F4` для commander-style створення нового файла у поточній панелі;
- `Shift+F5` для копії з новим іменем у тій самій панелі;
- `Shift+F6` для inline rename;
- `F7` для commander-style створення каталогу, включно з вкладеними шляхами, та опцією одразу перейти до нової папки;
- жива нижня F-панель, яка змінює підписи для `Shift+F4`, `Shift+F5`, `Shift+F6`, `Alt+F4`;
- налаштовувані шрифти типових діалогів з окремими ролями `header/body/emphasis/caption` і живим preview у `Options`;
- налаштування мови інтерфейсу з режимом `Auto` або явним вибором `EN/DE/UK`;
- опційне автооновлення панелі при зовнішніх змінах у відкритому каталозі;
- збереження положення, розміру і maximized-стану вікна з перевіркою на актуальну конфігурацію моніторів;
- локалізація `EN`, `DE`, `UK`;
- централізоване логування помилок поруч із користувацьким конфігом.

## Основні компоненти

Ключові частини проєкту:

- `Application/` — головне вікно та координація commander-команд;
- `Browsers/` — базовий контракт, фізична файлова панель, archive- і GEDCOM-провайдери;
- `Controls/` — breadcrumb-навігація та Quick View-контроли;
- `Dialogs/Common/`, `Dialogs/Operations/`, `Dialogs/Legacy/` — службові, операційні та сумісні legacy-діалоги;
- `Editors/Text/`, `Editors/RichText/` — внутрішні текстовий і RTF-редактори разом із власними ресурсами;
- `Viewers/Csv/`, `Viewers/Compare/`, `Viewers/Images/` — CSV viewer, порівняння та image/EXIF viewer із залежними файлами;
- `Services/` — архіви, файлова класифікація, іконки, операції та спільний стиль UI;
- `Infrastructure/` — логування, performance trace, локалізація, settings і платформний код.

## Гарячі клавіші

- `F2` / `Ctrl+Q` — `Quick View`
- `F3` — перегляд
- `Shift+F3` — порівняння з відповідником у пасивній панелі
- `F4` — редагування
- `Ctrl+R` — перечитати активну панель
- `Shift+F4` — створити новий файл у активній панелі
- `Ctrl+N` — створити новий файл через системний діалог
- `F5` — копіювати в пасивну панель
- у відкритому архіві `F5` — розпакувати вибрані записи до пасивної файлової панелі
- `Ctrl+PgDn` — відкрити `.ged` як GEDCOM-панель або перевірити сигнатуру файла й увійти в нього, якщо це ZIP/TAR/TAR.GZ, навіть із нетиповим розширенням
- `Backspace` — повернути попередній список панелі, включно з переходом назад на інший диск або в архів
- `Alt+Up` — перейти строго до батьківського каталогу та виділити каталог, з якого виконано повернення
- `Shift+F5` — створити копію в тій самій панелі
- `Alt+F5` — запакувати виділені елементи в архів
- `F6` — перемістити в пасивну панель
- `Shift+F6` — перейменувати
- `F7` — нова папка
- `F8` — видалити
- `Alt+F9` — розпакувати вибраний архів
- `Ctrl+L` — перейти до командного рядка
- `Ctrl+Enter` — вставити в командний рядок назву елемента під курсором
- `Enter` у командному рядку — виконати команду в каталозі активної панелі
- `Up` / `Down` у командному рядку — переглянути історію команд
- `Shift+Enter` — запустити вибраний `.bat/.cmd/.exe/.com/.ps1` або введену команду в консолі, що не закривається

## Збірка

### Вимоги
- Windows
- .NET SDK 8+

### Команда
- `dotnet build Commander.NET.csproj`

Активна збірка орієнтована на `AnyCPU/x64` і більше не залежить від `x86-only` `IWshRuntimeLibrary`.
